In my opinion, the first line shows the direct cause. Many errors with different root causes have the same direct cause. Showing this situation to developer and openQA users will be helpful and the root causes are less important. Thus, I focus on the first line.

To measure the similarity of error messages, I chose Levenshtein distance.

Levenshtein distance

Levenshtein distance is a metric for measuring the difference between two strings. It is name by the Soviet mathematician Vladimir Levenshtein in 1965. Levenshtein distance measures the distance from a string to another string. The algorithm of calculating Levenshtein distance is: \[lev(a,b)=\begin{cases} |a|, &if\ |b|=0\\ |b|, &if\ |a|=0\\ lev(tail(a),tail(b)), &if\ a[0]=b[0]\\ 1+min \lbrace lev(tail(a),b),lev(a,tail(b)),lev(tail(a),tail(b))\rbrace, &otherwise \end{cases}\]

where $ lev(a,b) $ indicates the Levenshtein distance of string \(a\) and \(b\) ; $ tail(a) $ indicates the substring which removes the first character of \(a\).

As the above formula shows, recursive implementation can be used to calculate Levenshtein distance. However, it may cost unnecessary time and space to calucate repeated substructures. Dynamic programming is a better choice. Of course, there are existing packages to use.
